# ReferenceFootnote 0.5.0-dev 规划方案

> 规划日期：2026-05-18
> 规划来源：用户指出当前 0.4.0-dev 核心流程倒装，要求重新规划 retrieval-first 管线
> 分支：`codex/0.5.0-dev-retrieval-first`

---

## 一、规划结论

### 1.1 核心判断：0.4.0-dev 的流程是"倒装"的

0.4.0-dev 的实际阶段机是：

```
A3 引用需求诊断 → A4 RAG 反查(对空库或残库做) → A6 证据映射 → A7 补库(只补 gap) → …
```

这个流程有三个致命问题：

1. **空库 RAG 反查是浪费或误导。** 如果文章是全新的、没有已建文献库，A4 RAG 反查结果全是 `no_support`——纯属浪费。更危险的是，如果 RAG 偶然命中几篇不相关文献，它们会被当成"已有证据"进入候选池，绕过正式检索入库的质量门槛。
2. **补库只补 gap，不做系统文献库建设。** 当前 A7 `build-search-handoff.py` 只处理 `critical_gaps`，不做文章级的系统文献库建设。结果是：即使补了几篇，文献池仍然是碎片化的，无法支撑 25-30 篇参考文献的目标。
3. **没有"文章反推检索蓝图"阶段。** 关键词和检索方向完全依赖逐条 claim 的 `search_dimensions_for_text()`，而不是从文章整体研究方向、理论框架、政策背景系统地推导。

### 1.2 正确流程

用户明确的正确流程是：

1. 读取已写文章。
2. 从文章内容反推出研究方向、证据需求、关键词、中英文检索词、理论线索、来源类型。
3. 先调用/交接给 `检索入库` skill，完成足够数量和质量的文献检索、PDF 获取、Zotero 保存、RAG 导库。
4. 等 `检索入库` 返回结构化入库完成状态。
5. 再通过 RAG 对文章 claim 逐步反查、比对、筛选真正应该使用的参考文献和脚注。
6. 最终输出脚注/参考文献插入计划、问题风险清单、补库缺口清单和交给正文写作/人工的任务包。

### 1.3 0.5.0-dev 的核心改动

**将"文章反推检索蓝图"和"初始文献库建设交接"前移到 RAG 反查之前。**

新流程总览：

```
A0-A3 (不变)
→ A3.5 检索蓝图
→ A4   初始文献库建设交接
→ A4.5 初始文献库调用包
→ A5   等待/应用入库完成
→ A5.5 入库完成质量验收
→ A6   入库后 RAG 反查请求
→ A6.5 RAG 响应验证/解释
→ A7   证据映射
→ A7.5 缺口二轮补库交接
→ A7.6 缺口补库调用包
→ A8   二轮入库完成
→ A8.5 二轮 RAG 反查
→ A9a-A9c 候选池/裁剪
→ A9   脚注方案
→ A10-A10c 质量门禁/真实性复核
→ A11  交付
```

### 1.4 三类请求的明确区分

| 类别 | request_type | 触发时机 | 交接目标 | macro_round | target_kb |
|------|-------------|---------|---------|-------------|-----------|
| 初始文献库建设 | `search_intake_library_build` | A3.5 检索蓝图生成后 | 检索入库 | round1 | B |
| RAG 后发现缺口的二轮补库 | `search_intake_gap` | A7 证据映射发现 critical_gaps 后 | 检索入库 | round2 | C |
| 正文写作/人工补注任务包 | N/A（delivery 输出） | A11 交付时 | 正文写作 / 人工 | N/A | N/A |

## 九、初始文献库建设质量门槛详述

### 9.1 目标文献池数量

| 指标 | 目标值 | 最低通过值 | 说明 |
|------|--------|-----------|------|
| 初始池总数 | 40 篇 | 24 篇（40×0.6） | 低于 24 篇为 blocking |
| 最终参考文献目标 | 25-30 篇 | 20 篇 | 低于 20 篇为 warning，低于 15 篇为 blocking |

### 9.2 可消费正文材料

| 指标 | 门槛 | 状态 |
|------|------|------|
| 池均值（pool_avg_usable_text_chars） | ≥200 字/篇 | 低于 200 为 blocking |
| 单篇标记 | ≥200 → `normal`；100-199 → `below_average`；<100 → `very_low` | very_low 条目进入人工复核 |

### 9.3 文献类型覆盖

| 类型 | 目标数量 | 说明 |
|------|---------|------|
| 中文核心期刊论文 | ≥12 篇 | 理论和实证基础 |
| 正式专著/专著章节 | ≥5 篇 | 深度理论支撑 |
| 政策文件/法规/标准 | ≥5 篇 | 政策类 claim 支撑 |
| 一手材料/案例/档案 | ≥3 篇 | 实证/事实 claim 支撑 |
| 英文论文 | ≥5 篇 | 国际比较和理论锚点 |
| 经典/奠基文献 | ≥3 篇 | 学术史定位 |
| 专利/技术标准 | 按需 | 技术类文章必需 |

类型覆盖最低要求：至少覆盖上述 7 类中的 3 类（`type_coverage_minimum: 3`）。

### 9.4 RAG 入库率

| 指标 | 门槛 | 状态 |
|------|------|------|
| rag_indexed_count / total_sources_ingested | ≥50% | 低于 50% 为 warning |
