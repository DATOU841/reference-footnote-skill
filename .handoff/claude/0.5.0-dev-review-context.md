## 本轮问题

用户指出 0.4.0-dev 的核心流程倒装：不能先对空库/残库做 RAG，再从 no_support 中补库；正确做法是先根据文章反推检索方向和关键词，调用/交接 `检索入库` 建设足量文献库，再通过 RAG 对比筛选参考文献、脚注和风险点。

## 主要实现

- 新增 `scripts/build-search-blueprint.py`：从文章和 citation needs 生成检索蓝图、方向分桶、关键词、来源类型和文献池质量要求。
- 新增 `scripts/build-initial-search-handoff.py`：生成 `search_intake_library_build` 初始文献库建设请求。
- 新增 `scripts/validate-intake-quality.py`：检查初始库总数、池均可消费文本、来源类型覆盖、RAG indexed 比例。

- 修改 `scripts/stage-gate.py`、`scripts/build-delivery.py`、`scripts/build-evidence-map.py`、`scripts/startup.py`。
