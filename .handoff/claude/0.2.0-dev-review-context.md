## 本轮目标

修复 ReferenceFootnote 的协同能力：让它清楚、可验证地准备对 `检索入库` skill 的调用，以完成必要文献检索、Zotero/PDF/RAG 导库；并在 `检索入库` 返回入库完成状态后，准备补库后二轮 RAG 文献反查调用包。

## 已实现内容

- 新增 `scripts/build-search-intake-call.py`：读取 `search-intake-requests/<batch>.json`，生成交给 `检索入库` 的 JSON 调用包和中文提示词。
- 新增 `scripts/build-post-ingestion-rag-call.py`：读取 `intake-status.json` 和 `citation-needs.json`，只对 `import_status.rag_indexed=true` 的来源生成补库后二轮 RAG 调用包。
- 新增 `docs/collaboration-flow.md`：说明 ReferenceFootnote -> 检索入库 -> RAG 平台 -> ReferenceFootnote 的协同闭环。
- 新增 `references/rag-platform-interface.md`：说明补库前/补库后 RAG 反查输入输出协议。
- 更新 `SKILL.md`、`README.md`、`docs/state-machine.md`、公开介绍、CHANGELOG 和 RELEASE。
- 离线 fixture 从 16 个扩展到 18 个，新增 search-intake call package 和 post-ingestion RAG call package 场景。

## 边界

本轮没有运行真实 CNKI、WoS、Zotero、PDF 获取、RAG 导库、writing-pool、advance-pool、mimo 或服务器部署。

ReferenceFootnote 仍只准备调用包，不直接执行外部系统。真实检索/入库必须由 `检索入库` 在用户授权后执行；真实 RAG 查询必须由 RAG 平台或外部执行者返回结构化结果。
