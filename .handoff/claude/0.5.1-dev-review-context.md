# ReferenceFootnote 0.5.1-dev Claude 审查上下文

## 本轮目标

根据 Claude 的 `2026-05-20-rag-markdown-grounding-fix-plan.md` 修复 ReferenceFootnote 的证据核查路径：

- RAG 入库后已经由 MinerU/MU 或等价工具生成 Markdown/parsed text 并切片。
- ReferenceFootnote 不应默认要求回原 PDF 逐条核查。
- 默认链路应为：RAG chunk 定位 -> Markdown/parsed text 上下文核查 -> page_map 作为页码桥接 -> PDF 仅在 OCR、页码映射或复杂版式风险下 fallback。
- 相邻格具或近似教学研究只能标为 `analogy_only`，不得升格为直接支撑。
- 权属/产品来源缺一手材料时不得强行补注。

## 新增 fixture 覆盖

- full markdown path -> `full_markdown_grounding`
- chunk only -> `chunk_only_grounding`
- parsed text path -> `full_markdown_grounding`
- reliable page map -> `page_mapped_grounding`
- page map conflict -> `pdf_fallback_required`
- garbled source file but item key ok
- vertical text -> PDF fallback
- table evidence -> PDF fallback
- no ownership primary -> no forced insertion
- adjacent grid analogy -> `analogy_only` not direct support
