# ReferenceFootnote 0.5.3-dev Review Context

## Scope

This version fixes the S50 RAG reverse lookup responsibility boundary.

ReferenceFootnote still must not run CNKI, WoS, Zotero, PDF retrieval, or RAG ingestion. After `检索入库` completes 2.5 ingestion, ReferenceFootnote now controls read-only RAG reverse lookup by default and writes an internal executor response.

## Main Changes

- Version bumped to `0.5.3-dev`.
- Added `config/rag-executor.yaml`.
- Added `scripts/run-rag-reverse-lookup.py`.
- Changed S50 into:
  - S50a build request: `state/rag-requests/<batch>.json`
  - S50b execute reverse lookup: `state/rag-calls/<batch>.response.json`
  - S50c validate response: `state/evidence-interpretations/<batch>.json`
- `build-post-ingestion-rag-call.py` no longer marks RAG calls as `prepared_not_executed` or `requires_external_rag_operator`.
- `config/boundaries.yaml` keeps `rag_ingestion` forbidden and adds `rag_reverse_lookup_query` as allowed.
- Missing live RAG config now blocks with `missing_rag_executor_config`; it does not ask the user for a manual response.
- Offline tests now include mock executor, missing-config blocker, executor response validation, and boundary separation.
